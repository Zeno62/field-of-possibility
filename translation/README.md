Translation Workflow

翻译维护流程

This document defines how multilingual versions of Field of Possibility / 可能性网络 should be translated and maintained.

The Chinese edition is the source edition of this project.

⸻

1. Source Edition

The source edition is:

可能性网络白皮书 v4.0 正式版

Source language:

Simplified Chinese

Current source version:

v4.0.0

All translations should be based on the Chinese source edition unless a newer official Chinese source version is released.

⸻

2. Translation Principle

The goal of translation is not only linguistic accuracy.

The goal is to preserve:

* conceptual precision;
* philosophical structure;
* distinction between Awareness and Consciousness;
* distinction between Pure Possibility and Field of Possibility;
* distinction between manifestation and essence;
* the central mechanism of 认假为真;
* the non-religious and non-self-help boundary of the project.

Translations should avoid turning the project into:

* a religion;
* a mystical doctrine;
* a self-help system;
* a success philosophy;
* a quantum mysticism framework;
* a personal belief system.

⸻

3. Translation Status

Each translated file should clearly indicate its status.

Recommended statuses:

* draft — initial translation, not yet reviewed;
* reviewed — reviewed by at least one contributor;
* official — accepted as an official translation;
* outdated — source edition has changed and translation needs updating.

Example metadata:

language: en
source_language: zh-Hans
source_version: v4.0.0
translation_status: draft
review_status: unreviewed

⸻

4. Version Relationship

Chinese source version:

v4.0.0

English translation based on it should be marked as:

en-v4.0.0-draft

If the Chinese source changes to v4.1.0, translations based on v4.0.0 should be marked as outdated until reviewed.

⸻

5. Terminology

Before translating key concepts, check:

glossary/translation-map.md

Key terms include:

* 纯粹可能性
* 觉
* 明性
* 可能性母网
* 全知
* 全量
* 意识
* 权重
* 个业
* 共业
* 认假为真
* 单帧中断
* 回归
* 假我

If a term is difficult to translate, open a Discussion before changing the translation map.

⸻

6. Contribution Path

Use:

* Discussions for terminology debates and translation questions;
* Issues for translation problems or unclear passages;
* Pull Requests for concrete translation edits.

Major terminology changes should be discussed before being submitted as Pull Requests.

⸻

7. Translation Boundary

This project welcomes careful translation and interpretation.

However, translation should not overwrite the source framework.

The English edition is an international-facing edition, but the Chinese edition remains the source of truth.
